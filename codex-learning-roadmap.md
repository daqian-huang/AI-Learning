# Codex 个人研究工作台学习路线

> 适用对象：非技术背景用户，关注 Bitcoin、黄金、美股研究，希望先搭建个人研究工作台，而不是一开始做复杂 App。

## 一句话目标

用 30 天把 Codex、GitHub、Markdown 和 AI 提问模板组合起来，建立一个能长期保存资料、整理观点、辅助生成每周复盘报告的 BTC / 黄金 / 美股研究资料库。

## 学习顺序总览

建议按照下面 6 个阶段学习：

1. 新手入门
2. 资料管理
3. 研究工作台
4. 自动化
5. App / Agent 开发
6. 变现案例

这个顺序的核心逻辑是：先会用，再会整理；先做自己的研究资料库，再考虑自动化；最后才考虑 App、Agent 和变现。

5. 学习 Codex 最简单方法，搜索主页所有 Codex 推文  
   https://x.com/gengdaJ/status/2063147147223355740

### 重点学什么

- Codex 是什么：可以理解成一个能看文件、改文件、运行命令的 AI 助手。
- GitHub 是什么：可以理解成你的研究资料云端档案柜。
- 仓库是什么：一个专门放某类资料的文件夹，例如“我的市场研究”。
- Markdown 是什么：一种简单写笔记的格式，不需要学编程。
- 权限和安全：哪些文件可以让 Codex 看，哪些账号、密码、API Key 不能放进公开仓库。

### 先做什么练习

- 创建一个 GitHub 仓库，名字可以叫 `AI-Learning` 或 `market-research-workbench`。
- 建立 3 个文件夹：`bitcoin/`、`gold/`、`stocks/`。
- 每个文件夹里写一篇 `README.md`，用普通话说明“我关注什么”。
- 让 Codex 帮你把笔记改得更清晰，但不要让它替你做投资结论。

5. Codex + Obsidian + SQLite + 向量检索迭代升级  
   https://x.com/gengdaJ/status/2068555151733043504

### 重点学什么

- 先不用急着学 SQLite、向量检索这些技术词。
- 先学“资料如何分类”：新闻、数据、观点、复盘、待验证问题要分开放。
- 先学“文件命名”：让未来的自己一眼看懂这是什么资料。
- 先学“事实和观点分离”：事实是发生了什么，观点是你或别人怎么理解。

### 先做什么练习

- 在每个资产文件夹下建立 4 类笔记：
  - `news.md`：重要新闻
  - `data.md`：关键数据
  - `questions.md`：想继续验证的问题
  - `notes.md`：自己的观察
- 练习让 Codex 把一篇文章整理成 4 部分：
  - 事实
  - 影响
  - 风险
  - 我还需要验证什么
- 建立一个 `prompts/` 文件夹，保存常用 AI 提问模板。

### 推荐先读

   https://x.com/gengdaJ/status/2054928950783299838

4. Claude Fable5 vs Codex，坚持使用 Codex 理由  
   https://x.com/gengdaJ/status/2064503056474222925

### 重点学什么

- 研究工作台不是复杂 App，而是一套固定流程。
- 你的工作台应该回答 5 个问题：
  1. 我今天收集了什么？
  2. 这些信息属于 BTC、黄金还是美股？
  3. 哪些是事实，哪些是观点？
  4. 本周最重要的变化是什么？
  5. 我下周要验证什么？

### 先做什么练习

- 建立 `templates/weekly-market-report-template.md`，作为每周复盘模板。
- 建立 `reports/` 文件夹，每周写一篇报告，例如 `2026-W26.md`。
- 让 Codex 根据 `bitcoin/`、`gold/`、`stocks/` 的笔记生成周报草稿。
- 你只负责三件事：检查事实、删掉夸张结论、补充自己的判断。

### 重点学什么

- 自动化不是一开始就写程序，而是把重复动作变成清单。
- 先自动化“整理文字”，再自动化“抓取数据”。
- 对金融研究来说，自动化输出必须保留来源，不能只给结论。

### 先做什么练习

- 让 Codex 每周自动检查：三个资产文件夹是否都有本周更新。
- 让 Codex 根据模板生成周报初稿。
- 让 Codex 输出“本周缺失的信息清单”。
- 等流程稳定后，再考虑自动抓取网页、行情或新闻。

### 重点学什么

- Agent 可以理解成“有固定任务和固定流程的 AI 助手”。
- App 可以理解成“把你的工作流做成别人也能点按钮使用的工具”。
- 你现阶段不需要马上开发 App。先把自己的研究流程跑顺，再决定哪些部分值得产品化。

### 先做什么练习

- 设计 3 个“虚拟助手”：
  - BTC 助手：整理 BTC 新闻、ETF、链上和宏观流动性。
  - 黄金助手：整理美元、美债收益率、实际利率、通胀和避险需求。
  - 美股助手：整理财报、估值、行业趋势和市场情绪。
- 给每个助手写一段任务说明，不要求写代码。
- 让 Codex 模拟这些助手分别输出本周重点。

# 30 天学习计划

## 第 1 周：搭好资料库骨架

### Day 1：明确目标

- 写下你为什么要做这个研究工作台。
- 明确三类研究对象：Bitcoin、黄金、美股。
- 输出：`research-system.md` 初版。

### Day 2：建立文件夹

- 建立 `bitcoin/`、`gold/`、`stocks/`、`reports/`、`templates/`、`prompts/`。
- 每个文件夹写一句用途说明。
- 输出：一个能看懂的资料库结构。

### Day 3：学习 Markdown

- 学会标题、列表、表格、引用这 4 个常用格式。
- 不需要学习复杂语法。
- 输出：每个资产一篇基础笔记。

### Day 4：建立 BTC 笔记

- 记录 BTC 的关键观察维度：价格趋势、ETF 资金流、链上数据、宏观流动性。
- 输出：`bitcoin/README.md`。

### Day 5：建立黄金笔记

- 记录黄金的关键观察维度：美元指数、美债收益率、实际利率、通胀、避险需求。
- 输出：`gold/README.md`。

### Day 6：建立美股笔记

- 记录美股的关键观察维度：公司基本面、财报、估值、行业趋势、市场情绪。
- 输出：`stocks/README.md`。

### Day 7：第一次周复盘

- 不追求专业，只回答：这周我看到了什么？我还不懂什么？
- 输出：`reports/week-01-start.md`。

## 第 2 周：形成资料整理习惯

### Day 8：建立新闻整理模板

- 让 AI 把新闻整理成：事实、影响、风险、待验证问题。
- 输出：`prompts/news-summary.md`。

### Day 9：整理 3 条 BTC 信息

- 每条信息都写来源、日期、事实、你的理解。
- 输出：BTC 新闻笔记更新。

### Day 10：整理 3 条黄金信息

- 重点练习区分“利多黄金”和“利空黄金”的原因。
- 输出：黄金新闻笔记更新。

### Day 11：整理 3 条美股信息

- 重点练习区分公司新闻、行业新闻、宏观新闻。
- 输出：美股新闻笔记更新。

### Day 12：建立问题清单

- 每个资产写 3 个你还没搞懂的问题。
- 输出：`questions.md` 或对应问题区块。

### Day 13：让 Codex 帮你检查结构

- 让 Codex 看仓库结构，指出哪些文件命名不清楚。
- 输出：整理后的资料库。

### Day 14：第二次周复盘

- 用模板写一篇周报。
- 输出：`reports/week-02-review.md`。

## 第 3 周：搭建研究工作台流程

### Day 15：建立周报模板

- 周报包含：本周摘要、BTC、黄金、美股、风险、下周关注。
- 输出：`templates/weekly-market-report-template.md`。

### Day 16：练习让 AI 生成周报草稿

- 输入本周笔记，让 Codex 输出草稿。
- 你负责检查和修改。
- 输出：一篇 AI 辅助周报。

### Day 17：建立事实检查清单

- 每个结论都检查：来源、日期、数据口径、反方观点。
- 输出：`prompts/fact-checklist.md`。

### Day 18：建立资产对比表

- 用表格比较 BTC、黄金、美股当前主要驱动因素。
- 输出：一张每周更新的对比表。

### Day 19：建立“本周最重要问题”

- 每周只选 3 个最重要问题，避免信息过载。
- 输出：本周关注问题清单。

### Day 20：模拟投研会议

- 让 Codex 扮演质疑者，挑战你的判断。
- 输出：反方观点记录。

### Day 21：第三次周复盘

- 把事实、观点、风险分开写。
- 输出：`reports/week-03-review.md`。

## 第 4 周：半自动生成每周复盘

### Day 22：设计固定周报流程

- 周一到周五收集资料，周六整理，周日复盘。
- 输出：你的固定周工作流。

### Day 23：写周报生成提示词

- 提示词要求 AI：不要编数据、标出缺失来源、保留不确定性。
- 输出：`prompts/weekly-report.md`。

### Day 24：测试周报生成

- 用前三周笔记生成一篇新周报。
- 输出：周报草稿。

### Day 25：建立缺失信息检查

- 让 Codex 告诉你：BTC、黄金、美股分别缺什么资料。
- 输出：缺失资料清单。

### Day 26：建立下周行动清单

- 每个资产只写 1 到 3 个行动项。
- 输出：下周研究计划。

### Day 27：整理个人提示词库

- 把好用的提示词保存到 `prompts/`。
- 输出：可重复使用的提示词库。

### Day 28：完成第四次周复盘

- 尝试让 Codex 先生成 70%，你人工修改 30%。
- 输出：`reports/week-04-review.md`。

### Day 29：回顾整个系统

- 删除没用的文件，合并重复模板。
- 输出：更干净的资料库。

### Day 30：确定下个月升级方向

- 三选一：
  1. 加强数据源整理。
  2. 加强自动化周报。
  3. 开始做简单网页或 App 原型。
- 输出：下个月计划。

# 给 Codex 的常用请求模板

## 整理资料

```text
请把下面资料整理成：事实、观点、可能影响、风险、待验证问题。请不要替我做投资建议，也不要编造数据。
```

## 生成周报

```text
请根据 bitcoin/、gold/、stocks/ 中本周更新的笔记，按照 templates/weekly-market-report-template.md 生成一份周报草稿。要求：
1. 区分事实和观点。
2. 标出缺少来源或需要验证的地方。
3. 不要给买卖建议。
4. 最后列出下周最值得关注的 3 个问题。
```

## 检查资料库

```text
请检查我的研究资料库结构，告诉我哪些文件命名不清楚、哪些内容重复、哪些地方缺少来源，并给出适合非技术用户的整理建议。
```
